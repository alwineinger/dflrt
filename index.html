What is 3 Loop Relay?

3 Loop Relay is a race-planning utility for team relay captains. Enter loop distances and each runner’s estimated pace; the app generates projected start and end times for every leg. As legs are completed, enter actual pace/time and projections update automatically. Legs can be reassigned to different runners, and the plan can be shared via the iOS share sheet (Messages, Mail, etc.).

Quick Start

  1. Initial setup: On first launch you’ll land on Settings. Set the Start Date & Time.
  2. Loops: Name and color each loop and set its distance (e.g., Green 4.7, Yellow 5.4, Red 5.8).
  3. Runners: Rename the default runners and set estimated paces (mm:ss).
  4. Start the race: Tap Start Race to generate the schedule.
  5. During the race: On Race Legs, tap a leg to enter actual pace/time; all projections recalc instantly.
  6. Reassign a leg: From Race Legs, tap a runner’s name to assign the leg to a different runner.
  7. Edit Settings: Tap Edit Settings to change loops/runners/legs. Use Apply Changes to re-apply settings to the plan.
  8. Share the plan: Tap the Share button on Race Legs to send via Messages, Mail, etc.

Sign-in

  • No account or sign-in required.
  • All data is stored locally on your device. No developer servers.

Permissions

  • No access requested for location, contacts, camera, microphone, Health, or background modes.
  • Uses Apple’s system share sheet only.

Sample data

Accept defaults for quick testing. You can rename runners, edit paces, add/delete runners, add/delete loops, change names/colors/distances, and set total legs. For point-to-point events, create a loop per stage and set total legs equal to number of stages.

FAQ

Does the app support more than three loops or eight runners?
Yes. Although it’s designed for 3-loop, 8-runner trail relays, you can adjust the number of loops, runners, and total legs to fit any format, including point-to-point.

How do I reset everything?
Go to Settings → Reset to Defaults to restore default loops, runners, and legs and delete the stored plan.
